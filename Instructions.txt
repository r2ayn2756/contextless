#### **Setup**
```bash
mkdir backend && cd backend

npm install -D nodemon
```

#### **Environment Variables** (`.env`)
```
ANTHROPIC_API_KEY=your_key_here
PORT=3001
```

#### **server.js Structure**
```javascript
- Express app setup with CORS
- Body parser (JSON, limit: 50mb for large text)
- Error handling middleware
- Rate limiting (optional but recommended)
- Routes mounted at /api
```

#### **Routes: compress.js**

**POST /api/compress** endpoint must:
1. Validate input (max 5M characters)
2. Determine if chunking needed (chunk if >100K chars)
3. Call Claude API with optimization prompt
4. Return structured response

**Chunking Logic** (`utils/textChunker.js`):
- Split text at sentence boundaries
- Target chunk size: 50K characters
- Maintain context overlap (500 chars) between chunks
- Function: `chunkText(text, maxChunkSize = 50000)`

**Claude Processing** (`utils/claudeProcessor.js`):
```javascript
// For each chunk or full text, send this prompt structure:

const systemPrompt = `You are a context optimization expert. Your job is to extract ONLY the essential information from the provided text, removing redundancy, filler words, and unnecessary details while preserving all key facts, data points, and actionable information.

Rules:
- Preserve ALL important facts, numbers, names, and dates
- Remove conversational filler and repetition
- Keep technical terms and specific details
- Maintain logical flow
- Output should be 60-80% shorter than input
- Use clear, concise language`;

const userPrompt = `Optimize this text for AI consumption:

${chunkText}

Provide:
1. A brief summary (2-3 sentences)
2. The optimized content with only essential information`;
```

**Error Handling**:
- Catch API errors and return user-friendly messages
- Handle rate limits gracefully
- Validate API key on startup
- Log errors without exposing sensitive data

**Token Counting**:
- Approximate: `tokens â‰ˆ characters / 4`
- Return original and optimized counts
- Calculate compression ratio
